# WebP Lab Deployment Notes

This repository is deployed on the OMV host `IsmaelOMV` at `192.168.8.124`.

## Production topology

- The app runs in Docker on the OMV host.
- `nginx` on the OMV host proxies `http://<host>/webp-lab/` to the container on `127.0.0.1:3001`.
- The Next.js app is built with `NEXT_PUBLIC_BASE_PATH=/webp-lab`, so absolute client fetches must always go through `withBasePath(...)`.
- LAN access is expected to work at `http://192.168.8.124/webp-lab/`.
- Tailscale access is expected to work at `http://ismaelomv.tail882371.ts.net/webp-lab/`.

## Source of truth and auto-updates

- GitHub is the source of truth: `https://github.com/Ismaeliki11/WebP-Lab.git`.
- The OMV host keeps a deployment clone under `/docker-compose/webp-lab/repo`.
- A systemd timer runs `/usr/local/bin/webp-lab-autodeploy` every minute.
- The timer fetches `origin/master`, rebuilds the container only when a new commit is detected, and keeps the existing container running if the rebuild fails.
- Auto-update is pull-based. From the user's point of view it updates automatically after a push without storing GitHub secrets on the server.

## Host paths on OMV

- Deployment root: `/docker-compose/webp-lab`
- Git clone: `/docker-compose/webp-lab/repo`
- Runtime env file: `/docker-compose/webp-lab/runtime/webp-lab.env`
- Local rembg model cache: `/docker-compose/webp-lab/runtime/rembg-models`
- Nginx include: `/etc/nginx/openmediavault-webgui.d/webp-lab.conf`
- Systemd files:
  - `/etc/systemd/system/webp-lab-autodeploy.service`
  - `/etc/systemd/system/webp-lab-autodeploy.timer`

## Runtime decisions

- Background removal is expected to run locally through `rembg` inside the container unless `REMOVE_BG_API_KEY` is set.
- HEIC support relies on Linux `libheif` tools in the container: `heif-enc` for encoding and `heif-convert` or `heif-dec` for decoding.
- Large uploads are intentional. Nginx is configured with `client_max_body_size 2G`.
- The production container uses `NODE_OPTIONS=--max-old-space-size=8192` unless overridden in the runtime env file.

## Rules for future edits

- Do not edit `/etc/nginx/sites-enabled/openmediavault-webgui` directly on OMV. It is auto-generated by OMV. Use `/etc/nginx/openmediavault-webgui.d/webp-lab.conf`.
- Keep production-only values out of the git repo. Persist them in `/docker-compose/webp-lab/runtime/webp-lab.env`.
- If new client-side network calls are added, wrap paths with `withBasePath(...)` so the app keeps working under `/webp-lab`.
- If the deployment branch changes from `master`, update the host auto-deploy script as well.
